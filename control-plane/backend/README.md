# Backend

FastAPI control plane for multi-host btop monitoring.

## Scope

- Web login and session token issuance
- API key creation for monitoring agents
- Agent inventory and latest metrics
- Historical metrics in SQLite
- Per-user default dashboard layout

## Quick start

```bash
cd /Volumes/Code/btop/control-plane/backend
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 9000
```

Environment variables are documented in `.env.example`.

## Docker and deployment

For the bundled image that serves the built frontend and keeps SQLite on a persistent volume, use the `control-plane/` deployment files:

```bash
cd /Volumes/Code/btop/control-plane
docker compose up --build -d app
```

For containerized hot-reload development:

```bash
cd /Volumes/Code/btop/control-plane
docker compose --profile dev up --build backend-dev frontend-dev
```

See `../docs/DEPLOYMENT.md` for the full container layout, environment variables, and agent routing details.

## Auth model

- Web UI users authenticate with username/password and receive a bearer token.
- Agents authenticate with backend-issued API keys.
- The current C++ agent stores that key in `auth_token` and sends `Authorization: Bearer <api_key>` to the compatibility routes under `/api/*`.
- The backend also accepts `X-API-Key` for non-C++ agents and future clients.
- API keys are generated by the backend and only shown once at creation time.

## Operator runbook

1. Start the backend:

```bash
cd /Volumes/Code/btop/control-plane/backend
conda run -n base uvicorn app.main:app --host 0.0.0.0 --port 9000
```

2. Sign in with the bootstrap admin from `.env.example` unless you overrode it:
   - username: `admin`
   - password: `admin123456`

3. Create an API key in the Web UI or with the API after login.

4. Put that key into the C++ agent config as `auth_token`.

5. Point the agent at the backend host and port:
   - native backend quick start: `server_port: 9000`
   - bundled Docker runtime: `server_port: 9000`

See [DEPLOYMENT.md](/Volumes/Code/btop/control-plane/docs/DEPLOYMENT.md) and [distributed-client.example.json](/Volumes/Code/btop/control-plane/deploy/distributed-client.example.json) for the agent sample config and deployment-specific routing.
