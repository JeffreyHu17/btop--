# Refactor Direction

## Target Architecture
- `btop-agent` stays in C++ and only does local metrics collection + upload.
- `control-plane/backend/` becomes the product-facing FastAPI service.
- `control-plane/frontend/` becomes the Vite + React management console.

## Why
The requirements now include:
- human users
- API key issuance
- dashboard customization
- chart-heavy web UI
- Docker deployment

Those are web-platform concerns. They are a poor fit for the experimental C++ HTTP server, but a good fit for FastAPI + SQLite initially.

## Security Model
- Human users authenticate with username/password and receive an access token.
- Agents do not use human tokens.
- API keys are generated by the backend, hashed at rest, and shown only once to the operator.
- Frontend only triggers key creation; it does not mint trusted keys locally.

## Storage Model
Initial backend tables:
- `user`
- `apikey`
- `agent`
- `metricsample`
- `dashboard`

## Migration Path
1. Keep the current C++ agent.
2. Keep FastAPI compatibility routes under `/api/*` for the current agent, or add a configurable API prefix before removing them.
3. Move web UI development to `control-plane/frontend/`.
4. Retire the C++ web server once feature parity is reached.
